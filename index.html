Spotify Server Documentacion

Este projeto foi desenvolvido no quinto módulo do curso de back-end da Blue EdTech com foco em aprimorar nossas habilidades em criar uma API que utilize TypeScript, NestJs, Prisma ORM e PostgreSQL. A presente API tem como objetivo que um usuário utilize de funcionalidades parecidas com as do Spotify e também possa criar perfis, dependendo do plano em que ele "assina". Artistas também tem suas próprias funcionalidades. Além disso também é possível que administradores possam gerir a plataforma.
Esta documentação visa apresentar como utilizar e testar nossa API mostranto exemplos de como criar um usuário, ser autenticado, criar perfis, playlists, etc.
Caso tenho algum prolema, dúvida ou sujestão fique a vontade para nos chamar no linkedin.

Autores: Lucas Marques e Gustavo Martins

Criando um usuário

Primeiramente comece criando um novo usuário para acessar a plataforma. Vá até 'user', e selecione a rota '/user/create' para informar os dados do usuário. Preencha todos os campos com dados válidos. Em 'userPlanId' você precisará informar um 'ID' de algum plano de usuário (os planos servem para limitar o uso de algumas funcionalidades da plataforma).

Vá até 'userPlanId' execute a rota GET '/user-plan', assim buscará todos os planos de usuário disponível. Então selecione, copie o 'ID' de algum plano e insira no campo 'userPlanId'. Execute a rota 'user/create' com todos os dados preenchidos.

Após criar um usuário é hora de fazer login. Insira seu email e senha.

Selecione e copie apenas o 'TOKEN', nele contém os dados do usuário criptografados.

Após copiar o 'TOKEN', vá até 'Authorize' no início da página. Agora cole seu token no campo 'Value' e clique em 'Authorize'. Agora você já está autenticado para continuar acessando as outras rotas da plataforma.

Criando um perfil para o usuário

Após estar autenticado, vá até 'profile' e na rota '/profile/create' insira um nome para o perfil e um link de uma imagem que será a imagem do perfil e execute a rota.

Na resposta da criação você verá o 'ID' do perfil criado, copie este 'ID', pois será necessário.

Agora vá até 'auth' e na rota 'auth/sing-in-profile' insira o 'ID' do perfil que você tinha copiado,e execute a rota. Assim será gerado um token JWT que contém as informações do perfil. Sempre quando este perfil for criar uma playlist, favoritar uma música, etc, não será necessário enviar novamente o id do perfil que está criando algo, a API pegará essa informação direto do token.
Pensando em como será o front-end e seguindo o modelo que a Netflix usa, após o usário fazer login, ele chegará na tela de seus perfis (rota GET 'profile/all'), então quando escolher o perfil em que quer entrar, executará esta rota 'auth/sing-in-profile' e após isso seria redirecionado para home page do seu perfil (rota GET '/').

Já com o 'TOKEN' na resposta, copie ele, pois será necessário alterar o token antigo para o novo.

Volte novamente em 'Authorize' no topo da página, clique em 'Logout' e insira o novo token e clique em 'Authorize'. Esse novo token contém as informações do usuário e também o id do perfil que está usando a plataforma.

Criando uma nova playlist

Para criar uma playlist vá até 'playlist' na rota '/playlist/create' e informe o nome da playlist, o link de uma imagem para a capa dela, e defina o campo 'private' como 'true' ou 'false'. Esse campo determina se a playlist vai ser privada, nenhum outro usuário possa acessar ela além de quem criou, ou pode ser uma playlist pública.
